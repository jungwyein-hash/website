# Apple 스타일 가이드 (apple.com 딥리서치 · 2024–2026)

> **사용법.** 사용자가 "애플스타일로 해줘"라고 하면 이 파일을 먼저 읽고 기준으로 삼는다.
> apple.com(iPhone / MacBook / AirPods 제품·스펙 페이지)의 현행 디자인 언어를 **UI · 디자인 · 인터렉션** 3축으로 정리했다.
> 각 항목은 구체 수치(px·em·ms·이징)를 우선한다. 끝에 **새미그룹 프로젝트 토큰/컴포넌트 매핑**과 **23항목 체크리스트**가 있다.
> 표기: `(검증)` = 라이브 페이지/HIG/티어다운으로 확인, `(관례)` = 인스펙션 수준 관례(픽셀 정확도 필요 시 DevTools 재확인).

---

## 1. 디자인 (Design)

### 1.1 컬러
- **중립 지배.** 캔버스는 거의 항상 순백(`#FFFFFF`) 또는 근흑(`#000000`). 회색은 보조 텍스트/구분선.
  - 1차 다크 텍스트 `#1d1d1f`, 2차 `#6e6e73`, 라이트 섹션 배경 `#f5f5f7` `(관례)`
- **색은 제품에서 나온다. 크롬(UI)에서 나오지 않는다.** 배경은 중립으로 두고, 화면에서 채도 높은 건 제품 사진뿐.
- **액센트 = 상호작용.** 브랜드 블루(`#0071e3`)는 거의 전적으로 링크/CTA 등 클릭 가능한 요소에만. "파랑 = 누를 수 있음"을 학습시킨다. 장식으로 쓰지 않는다.
- **섹션 교차 배경.** 흰색 ↔ 라이트그레이(`#f5f5f7`) 풀블리드 밴드를 번갈아 써 보더 없이 섹션을 구분.
- **사진은 중립 무봉제 배경** 위에 부드러운 접지 그림자. 복잡한 환경 없음.

### 1.2 타이포그래피
- **폰트:** San Francisco("SF Pro"). 옵티컬 패밀리 — ≥20px = **SF Pro Display**, ≤19px = **SF Pro Text**. (한글 대체: Pretendard/Apple SD Gothic)
- **사이즈 의존 트래킹(가장 많이 빠뜨리는 디테일).**
  - 대형 볼드 헤드라인(~34pt+): 살짝 **양수**(라틴 기준 +0.4px). **한글은 반대로 살짝 타이트**가 더 단단함.
  - 본문(17px): **음수 ≈ −0.02em(−0.43px)**
  - 큰 타입 → 타이트, 본문 → 약간 음수가 규칙.
- **타입 스케일:** 소수의 의도된 사이즈 + 큰 점프. 히어로 헤드라인 흔히 **48–96px**, 대형 뷰포트 **80–120px** `(관례)`.
- **굵기:** 헤드라인 Semibold(600)/Bold(700), 본문 Regular(400). 울트라신 거의 안 씀. 한 페이지 **2굵기 이내**.
- **행간:** 대형 헤드라인 **1.05–1.1**, 본문 **1.4–1.5** `(관례)`.

### 1.3 여백·그리드
- **여백이 곧 브랜드.** 한 화면에 한 아이디어. 헤드라인 위아래·제품 주변에 큰 여백. **보더가 아니라 여백으로 그룹화**.
- **8px 그리드.** 8/16/24/32/48/64/80. 섹션 세로 패딩 큼(피처 밴드 상하 **100px+**) `(관례)`.
- **중앙 정렬 좁은 텍스트 칼럼** 안에 풀블리드 미디어. 콘텐츠 컨테이너 폭 **~980px**(신규 밴드 1024–1200px) `(관례)`.

### 1.4 깊이
- 1px 보더만 쓰지 않고 **미세 그림자 레이어**로 입체. 카드/팝오버에 옅은 다중 그림자.

## 2. UI

### 2.1 레이아웃·내비게이션
- **글로벌 내비:** 얇음(~44–48px), **sticky**, 반투명 `backdrop-filter: blur()` — 콘텐츠가 밑으로 흐름 `(관례)`.
- 제품 페이지엔 그 아래 **로컬 서브내비**(제품명 + 점프 링크 + Buy pill)가 메인 내비 지나면 sticky.
- **풀블리드 미디어 + 중앙 컨테이너**(~980–1024px). 양쪽 여유 거터.

### 2.2 히어로 해부학
- 굵은 **제품명** + **효익 한 줄(benefit-first)** + **가격/Buy CTA 쌍** + **대형 중앙 제품 이미지/영상**.
- 실제 톤: "The thinnest iPhone ever. With the power of pro inside." / "Even more delightful. Even more durable."

### 2.3 제품 페이지 시퀀스 (검증: MacBook Pro)
1. 히어로 → 2. **Get the highlights**(스캔용 핵심 4–6개 불릿) → 3. Take a closer look(인터랙티브 뷰어) → 4. 성능/칩 → 5. AI → 6. 배터리 → 7. OS → 8. 생태계 → 9. 앱/유스케이스 → 10. 디스플레이→카메라→연결→보안 → 11. 업그레이드 비교 → 12. 구매 이유 → 13. 가치(환경·개인정보·접근성) → 14. 푸터.

### 2.4 스펙 표현
- **마케팅 페이지:** 스펙을 **효익으로 서술**("최대 24시간"), 표로 안 함.
- **가로 빅스펙 스트립:** 핵심 수치를 큰 숫자 + 작은 라벨, **세로 구분선**으로 가로 배열. (← 새미 "이달의 제품"에 적용한 패턴)
- **별도 `/specs/` 페이지:** 굵은 카테고리 헤딩(좌) + 라벨/값 행(우 들여쓰기), 모델 비교, 하단 각주 `[1]`~. 밀도 높고 장식색 없음.

### 2.5 버튼·카드·컴포넌트
- 버튼: pill 형태, 1차=솔리드 블루, 2차=아웃라인/고스트. 라운드 큼.
- 카드: 옅은 보더 + 미세 그림자. 호버 시 살짝 떠오름(translateY)·그림자 강화.
- 그룹화는 여백 우선, 구분선은 최소.

## 3. 인터렉션 (Interaction)

### 3.1 스크롤 구동 이미지 시퀀스 (시그니처)
- **기법:** 사전 렌더 **이미지 시퀀스(플립북)**를 `<canvas>`에 그림. `<video>` 아님 — canvas + `requestAnimationFrame()`이 스크럽에 매끈.
- **검증:** AirPods Pro = **148프레임**(`0001.jpg`…`0148.jpg`).
- **매핑:** `fraction = scrollTop / (scrollHeight - innerHeight)` → `frameIndex = floor(fraction * frameCount)`.
- 톨 컨테이너(`height: 500vh`)가 스크럽 구동, canvas는 `position: sticky` 세로 중앙, 전 프레임 **프리로드**, 느린 회선엔 정적 폴백.

### 3.2 핀/리빌 패턴
- **핀 미디어 + 스크롤 카피:** 미디어 `position: sticky; top: 50%; translateY(-50%)`, 레이어가 `clip-path: inset()`으로 와이프.
- **리빌 온 스크롤:** `opacity 0→1`, `translateY(20–40px → 0)`. 진입 시 페이드+슬라이드업.
- **모던 구현:** 네이티브 CSS `animation-timeline: view();` + `animation-range: entry 0% cover ~25%;` — 컴포지터에서 실행(JS 스크롤 핸들러 없음, 잰크 없음).

### 3.3 타이밍·이징
- **리빌/전환:** **0.6–1s**, ease-out 계열. 신뢰 가능한 애플 느낌 커브: 스티키 전환 `cubic-bezier(0.42,0,0.58,1)`, 페이드업 `cubic-bezier(0.25,0.1,0.25,1)` 계열 `(관례)`.
- **스크롤 스크럽:** `linear`(스크롤 위치와 1:1).
- 원칙: **콘텐츠 모션과 그래픽 모션 분리** — 텍스트는 가벼운 CSS, 무거운 제품 이미지는 canvas 스크럽.

### 3.4 호버·마이크로 인터랙션
- 미묘하게: 링크/CTA 색 심화, **언더라인 리빌**, 카드 **scale ~1.02–1.05**, 셰브론 넛지. 빠르게 **150–250ms ease**. 바운시/장난스러움 없음.

### 3.5 비디오
- **autoplay·muted·loop·inline·playsinline** 풀블리드 배경/히어로 영상. 접근성 위해 재생/정지 컨트롤 동반. 모션이 제품을 파는 곳에만.

### 3.6 접근성
- `prefers-reduced-motion: reduce` 시 스크롤 구동 애니 비활성, 콘텐츠 즉시 표시. GPU 친화(transform/opacity만).

## 5. 체크리스트 — "애플스럽나?" (23점, ≥19 강함 / 14–18 근접)

**디자인** 1.백/근흑 배경 2.색은 제품에서 3.액센트는 상호작용만 4.SF/Pretendard 옵티컬 5.사이즈별 트래킹 6.헤드라인 ≥48px·600–700 7.굵기 ≤2·점프 스케일 8.넉넉한 여백·여백 그룹화 9.8px 그리드 10.중앙 컨테이너 ~980–1024px
**UI** 11.히어로=제품명+효익 한 줄+CTA+대형 미디어 12.Get the highlights 스캔 블록 13.피처=1아이디어+미디어 14.효익 서술 스펙 + 별도 상세표
**인터렉션** 15.스크롤 리빌(페이드+슬라이드, 0.6–1s ease-out) 16.핀/스크럽 시퀀스 17.GPU·`prefers-reduced-motion` 18.blur sticky 내비+로컬 서브내비 19.autoplay-muted-loop 배경 영상 20.호버 150–250ms 미묘
**카피** 21.짧고 단정·효익 우선(스펙 나열 금지) 22.리듬(대구·대조·운)·따뜻·무버즈워드 23.문단 벽·헤지·과장 없음
